Sublime编辑器使用

首先声明下，本文是为真正的编程小白而写。

Sublime是目前好评率比较高的开发编辑器，简洁、轻巧、美观、免费。
对比以前的 nodepad++, dreamweaver，我个人真的喜欢它的这些特点。给大家介绍下使用方法，仅仅针对web前端的简单开发。

我们一般使用的sublime都是英文界面，请不要感到迷茫或者害怕。很简单的一个小软件， 我们的英文词汇丝毫没有压力。

开始我们的教程：

1.新建网页

我们在桌面新建一个文件夹 a。

打开 sublime，ctrl＋n（苹果机则是 commond ＋ n），新建一个文件，这个文件没有任何的信息。

和普通的编辑器不一样的是，一般的编辑器在新建时可以指定文件类型，而 sublime 不是，我们在保存的时候才可以指定文件的类型。

这里我们保存到刚刚新建的a文件夹，指定文件类型为 html，或者直接输入文件名时，后缀名写上 .html，如 index.html

然后我们会发现，sublime 右下脚显示文件的类型由 Plan text 变成 HTML。

开始编写我们的 html 文件。输入以下的代码：
html 

<html>
<body>
        <p>submlime 编写的html代码</p>
 </body>
</html>

保存后，我们打开我们的 a 文件夹，找到 index.html，右键选择我们的浏览器打开。结果如图，我们打开了我们写的第一个web页面，很简单是吧！

但是，有个问题：中文显示的不正常。我们称之为中文乱码。

产生的原因就是我们的中文编码和英文不一样，要知道，计算机的世界里，英文和阿拉伯数字是王道，所以我们显示中文时要稍微的加点说明。我们给刚刚的代码添加一点东西，如下：

html 

<html>
<head>
    <meta charset="utf-8" />
 </head>
<body>
        <p>submlime 编写的html代码</p>
 </body>
</html>

然后保存文件，ctrl ＋ s (苹果机则是 commond ＋ n）。在浏览器中刷新我们刚刚的页面，就可以发现中文可以正常显示了。

2.新建样式文件、js文件

和新建网页一样，也是ctrl ＋ n，唯一不同的是，我们保存样式文件的时候，后缀名选择 .css，js文件的时候，后缀名选择 .js

对于web前端来说，最主要的文件就是这三类，如果熟悉使用sublime编辑器，我们的编写速度会很可观。
